# ネットワーク接続

このカテゴリの問題は最も発生頻度が高いですが、同時に最も解決しやすいものです：**原因の99%は「同じホットスポットに接続していない」か「ファイアウォールでブロックされている」ことです。**

:::tip 順番にトラブルシューティングしてみましょう（本当に効果があります）
1. **同じ Wi-Fi/ホットスポット**: 親機端（PC/タブレット）とスマホは必ず同じホットスポットに接続してください。
2. **Windows ファイアウォール**: 「摊盒/Booth-Kernel」の通信が許可されているか確認してください。
3. **ホットスポット再起動によるIPアドレス変更**: ホットスポットを再起動した後は、親機端で **QRコードを再生成** してください。
:::

## スマホでQRコードをスキャンした後、「接続タイムアウト」と表示されたり、ウェブページが開きませんか？

これは最も一般的な問題です。以下の順序で確認してください：

1. **ホットスポットを確認**: 親機（PC/タブレット）とQRコードをスキャンするスマホが**同じ** Wi-Fi/ホットスポットに接続されていることを確認します。
2. **ファイアウォールのブロックを確認**: Windowsの親機の場合、ファイアウォールが「摊盒」の通信を許可しているか確認するか、一時的にファイアウォールを無効にして試してみてください。
3. **IPアドレスの変更を確認**: ホットスポットを再起動した場合、IPアドレスが変更されている可能性があります。親機端で **「QRコードを再生成」** ボタンをクリックしてください。

:::warning ちょっとした経験則
イベント会場では、多くの人が無意識に会場の Wi-Fi に接続しがちですが、それは不安定で、デバイス間の通信を隔離する設定になっていることもあります。**ホットスポットによるローカルネットワークが最も安定します。**
:::

## インターネットがない / 電波が弱い場所でも使えますか？

**全く問題ありません。** Booth-Kernel は「オフライン優先」で設計されています。
一台のデバイスで **スマホのホットスポット** を有効にしてローカルネットワークを構築し、他のデバイスをそれに接続すればOKです：

- **インターネット接続は不要です**
- **モバイルデータ通信は消費しません**

## 学内ネットワーク / 公共Wi-Fiを使っていますが、デバイスが接続できません。

公共ネットワークでは通常 **AP隔離**（デバイス間の相互アクセスを禁止する設定）が有効になっています。
解決策はシンプルです：**必ずホットスポットを使用してネットワークを構築してください**。これがイベント会場で最も確実な方法です。

## 途中でネットワークが切断されました。データは失われますか？

**失われません。** 親機端（アプリ側）が閉じられていない限り、データは保持され続けます。
ネットワーク接続を回復した後、スマホ側でページを更新すれば、以前の状態に復元されます。
