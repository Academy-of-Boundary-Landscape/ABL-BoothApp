# 現場での運用

このページでは主に、決済QRコードの設置方法、注文の締め方、通知音、複数デバイスでの連携など、「現場で直面するであろう事柄」について回答します。

## 決済QRコードはどう設定しますか？ 複数の決済手段に対応していますか？

「イベント管理」で「編集」をクリックすると、画像をアップロードできます。
WeChat（微信）と Alipay（支付宝）の両方をサポートしている場合、2つのQRコードを**1枚の長い画像に合成して**アップロードするのが最も手軽です。

## 顧客が支払いを完了したら、システムは自動的に注文を完了しますか？

**しません。**
プライバシー保護と金融リスク回避のため、Booth-Kernelは決済インターフェースには接続していません。

「支付宝/微信支付から入金がありました」という通知を聞いた後、スタッフのスマホで手動で **「注文を完了」** をタップして、在庫を減らす必要があります。

## 新規注文があると音で通知されますか？

**はい、通知音が鳴ります。** 店員端（スタッフのスマホ）は、新規注文リクエストを受信すると通知音を再生します。

音が聞こえない場合は、まず以下を確認してください：

- スマホが**マナーモード/おやすみモード**になっていないか
- **メディア音量**が小さすぎないか

## 複数人 / 複数デバイスで同時に運営できますか？

可能です！ 同じホットスポットに接続していれば、以下のように運用できます：

- 「注文端末」としてタブレットを2台設置
- 3人のスタッフがそれぞれ「仕分け端末」としてスマホを手に持つ

すべてのデータはリアルタイムで同期されます。
